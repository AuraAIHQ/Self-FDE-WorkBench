## 0. 一句话

**AI 的下一站不是更聪明的模型，是把模型送进每一个真实场景。**
干这件事的人，过去叫 FDE；未来叫 **ANE — AI Native Engineer**。
这个仓库，是给 ANE 的工作台。

---

## 1. 从 FDE 到 ANE

### 为什么 FDE 突然火了

AI 已经成熟到必须走出「聊天框」和「开发者圈」，去服务真实的商业组织。这和当年 ERP 信息化的路径完全一样——技术成熟后，瓶颈从「能不能做」转移到「能不能落到具体业务里」。

FDE（Forward Deployed Engineer，前向部署工程师）这个词是 Palantir 在 2010 年前后发明的，本质是**把苦活脏活累活制度化**：你必须进到客户现场和内部，才能准确识别真实的业务逻辑与流程，才能提炼、优化，先给解决方案，再抽象为产品。

### ANE 的工作循环

```
企业/组织场景 ──► 痛点（反复沟通才能找到真正的卡点，客户必须深度参与）
                    │
                    ▼
              解决方案（结合组织工作流，持续迭代优化）
                    │
                    ▼
              抽象为产品（通用化、可复用）
                    ▲
                    │
        ┌───────────┴───────────┐
        │                       │
   已有 AI 能力            新增 / 创新能力
   （熟知架构、能力域、       · 已有能力的二次包装
     以及能力边界）           · 多种 AI 能力的组合
                             · 引入新资源维度的创新
```

抽象出的产品会反过来扩充 ANE 的能力库，形成正循环。

### ANE ≠ FDE

| | FDE | ANE |
|---|---|---|
| 出处 | Palantir, ~2010 | 本仓库提出 |
| 驻场 | **必须**驻场 | **不必**驻场 |
| 服务对象 | 大企业（付得起钱的） | 个体 / 中小组织 / 城市 |
| 依托 | 公司的内部方法论 | **开源 WorkBench + 数字公共物品** |
| 核心能力 | 业务理解 + 现场交付 | 业务理解 + AI 能力域调度 + 能力创新 |

**核心假设**：这个角色会长期存在——久到不该再叫 FDE。它是 AI Native Engineer。

## 2. 未来的三种角色

**前提推演**：如果 AI 把人类社会的生产力成本压到接近电费，每个人都要回答一个问题——**当我的岗位被 AI 取代，我还能干什么？**

**先说不会消失的**：炸油条的、面包店、一切服务业，永远存在。吃面包比打营养针效率低，但大部分人不会去打针——因为人有固有习惯和情感诉求，正是这些东西塑造了「你是个人」。放弃了这些，人味儿就没了。

**会被重构的是商业驱动的岗位**——那些为了极致效率而存在的岗位。在那里，只有三种角色留下核心价值：

### 🗣 Expresser · 表达者
AI 不会说「这件衣服真好看」「这家饭馆味道不错」。**因为它不是人。** 它表达不出人的情感。
→ KOL、博主、任何真诚建立自己 IP 与表达的人。有生存空间，有想象力，甚至能形成自己的经济闭环。

### 💡 Innovator · 创新者
当现有工具、协作方式、学习方式满足不了他想象中的那个更好的世界，他会自己动手把它搭出来。

> ChatGPT 出来后，OpenAI 没有做 coding agent；是 Boris（Anthropic）做了，并且扎根了数年。

**创新者与效仿者的区别，是对未来坚定的信心。** 全地图都是黑雾的时候，有人趟出一条路——这是引领作用，是对人类的贡献。效仿者能赚到钱，但没有创新者伟大。

创新不必宏大。只要是你自己的 idea——在 AI 加持下，从 idea 到产品，可能就是一个晚上。思考者、探索者，都属于这一类。

### 🔧 Builder · 建设者（= ANE）
站在创新者开辟的路上，用他们提供的更好的工具，去落地真实场景的细节分支，让社会变得更好。

小红书上很多女生、文科生，什么技术都不懂，但对某个困境有深入思考，就做出了很棒的产品。哪怕带一部分模仿和学习，完全没问题——**建设者要解决的，是和普通人、普通组织息息相关的小问题。而这些「小问题」，对具体的人来说，可能是每天都在承受的痛苦。**

### 三者可叠加
ANE 可能同时是创新者，也可能是个 YouTuber。这三种角色**泾渭不分明，是可以叠加的**。
希望每个人都能成为其中的一种，或多种。

## 3. 三个服务对象，三个操作系统

### 👤 个体 → **Sin90**

每个人每天都在跑同一个环，只是内容不同：

```
        ┌──────────────────────────────┐
        │                              │
        ▼                              │
   Information ──► Action ──► Connection ──► Communication
   （睁眼看表、微信、                    （连接人、工具、    （维系关系是为了
     小红书、拉开窗帘                     打电话、meet up）    沟通，沟通是为了
     看天气⋯⋯都是获取信息）                                   再获取信息）
```

个人的**生活、学习、社交、工作**都在这个环里逐步成长。
Sin90 是围绕这个环打造的 Apache 2.0 开源个体操作系统，目标是提升普通人的**数字福祉**。

### 🏢 社区 / 中小组织 / 商业组织 → **Cos72**

雇不起 FDE 的组织，能不能**自己 FDE**？把它拆散，通过协作去找到创新者和建设者。

组织变成 AI Native，核心是三件事：

1. **知识库 / KB** — 组织沉淀的知识，散落在数据库、老员工的脑子里、客户的印象里。AI 时代，它必须被沉淀下来。
2. **组织工作流** — 使命、愿景、价值观、商业逻辑，最终都要通过流程落地。公益组织和商业组织，一个道理。
3. **组织大脑** — 由两部分组成：
   - **大模型 API**（OpenAI / Anthropic / DeepSeek ⋯）
   - **自训练 LoRA**（阶段性结论，未来可能被打破）：用组织自身的大量数据做特定场景微调，让模型对这个组织了解得极其透彻。

外加 **Connectors / Sensors（各类 Agent）**：行业数据抓取与分析、供应商自动连接与报价、客户新品推送与售后⋯⋯它们喂养并支撑组织工作流。

### 🏙 城市 → **CityOS**

当每个组织都有了一个足够懂自己上下文的大脑，**组织之间的共享会涌现出新的协作模式**，以及城市内的资源聚合。

以城市为边界的协作 → 跨边界协作 → **吸引资源 + 吸纳人才**。
城市本身变成一个智能体，与所有人协作。

> 这件事没人做过。但只有先行者才享受得到数字福祉的红利。
> 清迈是旅游城市——它如何在全球旅游城市中吸引更多游客？这是市长、市政府乃至全体市民都希望被解答的命题。CityOS 就是去解答它的。
