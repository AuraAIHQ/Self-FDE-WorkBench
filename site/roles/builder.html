ROLE 03 / BUILDER = ANE

解决那些小问题——
它们是别人每天承受的痛苦

建设者站在创新者开辟的路上，用他们提供的更好的工具，去落地真实场景的细节分支。

小红书上很多女生、文科生，什么技术都不懂，但因为对某个困境有极深的思考，就做出了很棒的产品。 哪怕带一部分模仿和学习，完全没问题——建设者要解决的，本来就是和普通人、普通组织息息相关的小问题。 而这些「小问题」，对具体的人来说，可能是每天都在经受的痛苦。

定义

ANE — AI Native Engineer

FDE 这个词是 Palantir 在 2010 年前后发明的，本质是把苦活脏活累活制度化：你必须进到客户现场和内部，才能准确识别真实的业务逻辑与流程，才能提炼、优化，先给方案，再抽象为产品。

我们认为这个角色会长期存在——久到不该再叫 FDE。

FDEANE

方法

ANE 工作循环

四步，加一个能力飞轮。每一步都可能反复回退——尤其是找痛点那一步。

WorkBench 提供

全流程支持

不是课程，是工作台。你在真实场景里会卡在哪，我们就在哪。

TRACK 01

AI Native 选型

Model & Architecture

什么场景该用什么模型？闭源 API 还是开源自部署？能力边界在哪里，什么时候不该用 AI——这往往比「怎么用」更值钱。

TRACK 02

训练自己的小模型

LoRA / Fine-tuning

用组织自身的数据做特定场景微调，让模型对这个组织了解得极其透彻。这是一个阶段性结论，未来可能被打破——但现在，它有效。

TRACK 03

Memory 与 Context

Context Engineering

Memory 在 Context 管理里究竟该如何实践和落地？有哪些可选的技术方案？各自的代价是什么？

TRACK 04

Agent 与 Connector

Agents in Production

行业数据抓取、供应商自动连接与报价、客户新品推送与售后——把 Agent 接进组织的真实工作流，而不是做 Demo。

AuraAI 作为聚合 ANE 的组织，持续沉淀、交流、积累，从而帮助更多的人成为表达者、创新者和建设者。
