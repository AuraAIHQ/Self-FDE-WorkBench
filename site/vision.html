00

一句话

AI 的下一站不是更聪明的模型，是把模型送进每一个真实场景。

干这件事的人，过去叫 FDE；未来叫 ANE — AI Native Engineer。这个仓库，是给 ANE 的工作台。

01

从 FDE 到 ANE

为什么 FDE 突然火了

AI 已经成熟到必须走出「聊天框」和「开发者圈」，去服务真实的商业组织和其他组织。这是 AI 发展阶段的需求造成的——和当年 ERP 信息化的路程一模一样。

FDE（Forward Deployed Engineer）这个词是 Palantir 在 2010 年前后发明的。它的本质是把苦活脏活累活制度化：你必须到客户的现场和内部，才能准确识别真实的商业逻辑与流程，才能提炼、优化，先给出解决方案，再抽象为他们的产品。

ANE 的核心假设

我们认为这个角色会持续存在很久很久——久到它不该再叫 FDE。它是 AI Native Engineer（ANE）。它和 FDE 的核心区别是：不一定要驻场。

我们为 ANE 提供一个流程模型和相关的指导帮助，也就是这个 WorkBench，让他们能快速切入不同领域：个体、社区、商业组织、城市。

这个假设大概率是成立的。我们只是给了它一个新的名字，和一套新的工作流程。

02

前提推演

如果 AI 能让人类社会的生产力成本极度降低，以至于接近电费的价格，那所有人都要思考一个新问题：当我的岗位被 AI 取代，我究竟还能干什么？

这个预判没有严谨的科学论证，但它来自个人的经验观察和各方信息的反馈与分析。

先说不会消失的

炸油条的、面包店、一切服务业，一定会永远存在——几千年都是这样。

因为它不仅仅满足效率。我去吃一个面包，还有效率更高的方式——给你注射一支营养剂。那为什么大部分人都不会去打针？因为人带有固有的习惯和情感诉求，只有这种东西才塑造了「你是一个人」。当你把这些都放弃的时候，人味儿就消失了。

会被重构的是商业驱动的岗位

那些为了追求极致商业利益、无限提升效率（协作效率、生产效率）而存在的岗位。在那里，我们认为只有三种角色留下了核心价值。

03

三种角色

Expresser 表达者

AI 不会说「这件衣服很好看」「这个饭店味道不错」。AI 不会说的，因为它不是人，就这么简单。它表达不出人的情感，因为它不是人。

所以表达者会有很大的生存空间，或者说更多的想象力。KOL、以及任何积极建立自己 IP 与形象、去表达真实感受的人——未来会有自己的一席之地，甚至形成自己的经济闭环。

给表达者的工具 →

Innovator 创新者

当他有一个 idea，却发现当前的工具、社会环境、沟通方式、学习方式，无法满足自己预想的那种美好情形时——他会自己动手把这个产品、流程、协作搭出来。

非常明显的例子：大模型出来之后，ChatGPT 没有做 coding agent，是 Boris（Anthropic）去做了，而且扎根了数年。

创新者和效仿者的核心区别，是创新者对未来充满坚定的信心。很多人都没意识到这一点的时候，他已经开始披荆斩棘，为人类开辟新的道路。效仿者看到的是「原来做 coding agent 能赚很多钱」，他没看到全地图都是黑雾的时候有人趟出了一条路。效仿者能赚到钱，但他没有创新者伟大。

创新不必宏大，小一点的创新也没问题。只要是你自己的 idea——尤其在这个时代，在 AI 的加持下，从 idea 到产品，可能就是一个晚上的事。思考者、探索者，都是这一类角色的称呼。

给创新者的支持 →

Builder 建设者（= ANE）

有了创新者的引领，我们可以使用他们提供的更好的工具，去帮助更多实际场景落地。

小红书上有很多女生、文科生，什么都不懂，但因为对某个困境有很深入的思考，就做出了很棒的产品。他们包含了创新者的一些要素，但更多的是把前行的创新者提供的便利和基点作为起点，去建设细节的、分支的场景，让社会变得更美好。

哪怕有一部分效仿、模仿和学习，我觉得都没有问题。建设者本身要解决的，就是和普通人、普通组织息息相关的小问题。而这些小问题，对具体的人来说，可能是每天都在经受的痛苦。

成为 ANE →

三者可叠加

ANE 是不是创作者？可能他也是。是不是表达者？可能他也是个 YouTuber。这三者并不泾渭分明，相互之间是可以叠加的。

我希望每一个人，都能成为其中的一个角色，或多个角色。

04

三个操作系统

需求侧有三层：个体 / 中小组织 / 城市。对应三个 Apache 2.0 开源系统：Sin90、Cos72、CityOS。

个体每天都在跑一个环：Information → Action → Connection → Communication → 再回到 Information。生活、学习、社交、工作，全在这个环里。组织的核心则是三件事：知识库、组织工作流、组织大脑（大模型 API + 自训练 LoRA），外加各类 Connector/Agent。城市，则是让组织之间涌现出新的协作，最终城市本身变成一个智能体。

05

这个仓库做什么

当个体、中小组织、城市想让自己变成 AI Native 时，他们会去找谁？——找表达者、创新者、建设者。

而这个仓库站在中间，为双方提供数字公共物品：Sin90、Cos72、CityOS，以及一系列 WorkBench。

  • 给表达者：一键发布 16 个平台——对普通人这是个障碍，我们提供免费开源的仓库，帮你一步跨过去。不限于此。
  • 给创新者：实验与研究对象的对接。想在制造业做事？我们对接制造业的中小组织和企业主，让他们讲出痛点，你去落地。
  • 给建设者：全流程支持——AI Native 如何选型、如何训练自己的小模型、Memory 在 Context 管理中如何落地、有哪些可选技术方案。

AuraAI 作为聚合 ANE 的组织，持续沉淀、交流、积累，从而帮助更多的人成为表达者、创新者和建设者。
