DIGITAL PUBLIC GOODS / APACHE 2.0

三个操作系统
三层服务对象

个体、中小组织、城市。
它们不是产品，是数字公共物品——开源、免费、无许可，任何人都可以使用、修改和部署。

01 / 个体 · 设计中

Sin90 — 个体成长环

每个人每天都在跑同一个环，只是内容不同。睁开眼看表、刷微信、刷小红书——甚至只是拉开窗帘看看天气，都是获取信息，那是你的第一步。

Sin90 的目标只有一个：围绕这个环，为普通个体的工作、生活、学习乃至创业提供助力和加持，得到一些数字生活的福祉。

02 / 社区 · 中小组织 · 商业组织 · 进行中

Cos72 — 组织协作系统

很多中小组织没有研发力量、没有技术人员，甚至雇不起一个 FDE。那能不能自己 FDE？把它拆散，通过协作去找到创新者和建设者，用 ANE 的方式，建设自己组织的 AI 能力。我觉得是可以的。

CORE 01

知识库

Knowledge Base

组织沉淀的知识，散落在数据库里、老员工的脑子里、客户的印象里。在 AI 时代，它必须被沉淀下来。

CORE 02

组织工作流

Organizational Workflow

使命、愿景、价值观、商业逻辑——它为什么存在？所有这些软性的东西，最终都要通过流程落地。公益组织和商业组织，一个道理。

CORE 03

组织大脑

Organizational Brain

大模型 API（OpenAI / Anthropic / DeepSeek⋯）+ 自训练 LoRA：用组织自身的大量数据做特定场景微调，让模型对这个组织了解得极其透彻。

自训练 LoRA 是一个阶段性结论。这个结论未来可能被打破——但在今天，它让组织的流程效率提升得最多。

— 关于组织大脑

Connectors / Sensors（各类 Agent）

组织的工作流和个体一样，需要大量的连接器与传感器。它们本质上就是各种 Agent：

  • 行业数据的抓取、分析与新闻判断
  • 外部采购供应商之间的自动连接与报价
  • 对客户的定期新品推送与售后服务

这些 Agent 获取信息，并支撑你的组织工作流。

03 / 城市 · 远期

CityOS — 城市成为智能体

当每个组织都有了一个足够懂自己上下文的大脑，组织之间的共享，会涌现出新的协作模式，以及城市内的资源聚合。

EMERGE 01

组织间协作

Inter-Org Collaboration

不需要全知全能，只需要对彼此的信息和上下文有足够了解。新的协作模式会自己长出来。

EMERGE 02

吸引资源

Attract Resources

以城市为边界的协作，可以跨边界去吸引更多的外部资源。

EMERGE 03

吸纳人才

Attract Talent

城市本身变成一个智能体，与所有人协作——而不是与所有人博弈。

清迈是一座旅游城市。它如何在全球的旅游城市中吸引更多游客？我相信这是市长、市政府乃至全体市民都希望被解答的命题。CityOS 就是去解答它的。

这件事没有人尝试过。但只有先行者，才享受得到数字福祉的红利。
