# 交互流程与验收

> 客户：probe-chat ｜ 项目：probe-idea ｜ 交付物：app（web）
> 逐步交互 + 每步检查/验收标准
> 本文件由 FDE Copilot 随每轮对话自动维护。

> ⚠️ 本轮为【假设】初稿，供下游自动化测试消费。每步含「检查点」。

## 流程 A：团长发起团购（M0）
1. 团长进入 `/new` 发起页。
   - 检查：页面展示标题、截止时间、自提点、商品编辑区。
2. 填写标题、截止时间、自提点文本。
   - 检查：截止时间 < 当前时间时提交被拦截并提示。
3. 添加商品（名称、单价、单位；图片可选），可添加多个。
   - 检查：单价非正数或名称为空时该商品无法保存；商品数为 0 时不能发起。
4. 点击「发起」。
   - 检查：返回成功页，展示分享链接 + 二维码 + 团长管理链接（含 manager_key）。
   - 检查：管理链接与分享链接不同；分享链接不含 manager_key。

## 流程 B：居民参团下单（M0）
1. 居民打开分享链接 `/g/:id`。
   - 检查：展示商品列表、单价、截止倒计时、成团进度（若设目标）。
   - 检查：团购已截止时页面只读、下单按钮禁用并提示「已截止」。
2. 选择商品与数量。
   - 检查：全部数量为 0 时不能进入下一步；超过限购/库存时提示。
3. 填写姓名、楼栋门牌、手机号并提交。
   - 检查：手机号格式非法时拦截；必填项缺失时拦截。
   - 检查：提交成功后展示应付金额与自提信息。
4. 再次打开链接。
   - 检查：截止前能看到并修改本人订单（凭浏览器 token 或手机号查询）。

## 流程 C：团长汇总与备货（M0）
1. 团长打开管理链接的汇总页 `/g/:id/summary?key=`。
   - 检查：无 key 或 key 错误时拒绝访问。
2. 查看按商品统计。
   - 检查：各商品总数量与小计金额正确；应收总额 = 各订单金额之和。
3. 切换按居民视图。
   - 检查：每位居民的商品明细、应付金额、联系方式、自提点正确展示。
4. 导出清单。
   - 检查：导出 CSV/文本，中文不乱码，行数与订单数一致。
