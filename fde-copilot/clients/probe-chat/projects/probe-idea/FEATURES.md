## F1 发起/管理团购（团长）· M0
**用户故事**：作为团长，我想快速创建一次团购并录入商品，以便分享给居民下单。

**验收标准**
- Given 团长在发起页，When 填写团购标题、截止时间、自提点，并添加≥1个商品（名称、单价、单位，图片可选），Then 保存成功并生成唯一分享链接与二维码。
- Given 已发起的团购，When 团长编辑商品或截止时间，Then 更新对本次团购的新访问者生效。
- Given 团购已到截止时间或团长手动关闭，Then 该团购状态变为「已截止」，居民端不可再下单。

**边界/异常**
- 单价为负或空、截止时间早于当前时间 → 阻止保存并提示。
- 未添加任何商品 → 不允许发起。
- 已有订单后编辑商品价格 → 【待确认】是否影响已下单订单金额（默认：不影响历史订单，仅影响新订单）。

## F2 居民下单（参团者）· M0
**用户故事**：作为居民，我想选商品填数量并提交，以便参加这次团购。

**验收标准**
- Given 居民打开进行中的团购链接，Then 看到商品列表（名称/单价/单位/图片）、截止时间、成团进度。
- Given 居民选择商品与数量，When 填写必填联系信息（姓名、楼栋门牌、手机号）并提交，Then 生成订单并显示应付金额与自提信息。
- Given 居民再次打开链接，Then 能看到/修改自己刚下的订单（截止前）。【识别方式待确认，见 GAPS】

**边界/异常**
- 团购已截止 → 只读展示，禁止下单。
- 数量为 0 或全部商品未选 → 不允许提交。
- 超过限购/库存（若设置）→ 提示不可超买。
- 手机号格式校验。

## F3 订单汇总与统计（团长）· M0
**用户故事**：作为团长，我想一键看到全部订单和应收总额，以便备货和收款。

**验收标准**
- Given 团购有订单，When 团长打开汇总页，Then 看到：总订单数、按商品合计（各商品总数量、小计金额）、应收总额。
- Given 汇总页，Then 可切换「按居民」视图（每位居民买了什么、应付多少、联系方式、自提点）。
- Given 汇总页，When 团长导出，Then 得到可复制/下载的订单清单（CSV 或文本）。

**边界/异常**
- 无订单时显示空态。
- 导出编码需支持中文不乱码。

## F4 成团进度与截止（双方）· M0
**用户故事**：作为居民/团长，我想看到成团进度与倒计时，以便决定是否下单。

**验收标准**
- Given 团购设置了成团目标（份数或人数，可选），Then 展示当前进度（已达/目标）。
- Given 设置了截止时间，Then 展示倒计时；到点自动截止。

**边界/异常**
- 未设成团目标 → 只展示已下单份数，不显示进度条。
