# 交互流程与验收

> 客户：CMU Summer Mini Hackathon ｜ 项目：帮我做个团购小工具 ｜ 交付物：app（web）
> 逐步交互 + 每步检查/验收标准
> 本文件由 FDE Copilot 随每轮对话自动维护。

> ⚠️ v0 假设流程；每步"检查项"可直接转自动化测试用例。

## 流程 A · 发起人开团（对应 F1）
1. 打开首页 → 点"发起团购"。
   - 检查：出现表单，含商品名、单价、成团口径（人数/份数/金额三选一）、目标值、可选截止时间、可选取货点。
2. 填写并提交。
   - 检查：必填校验生效；单价/目标为正；截止时间不早于当前。
3. 创建成功 → 跳转团购页。
   - 检查：URL 含 groupId；页面显示商品信息、进度 0、"复制链接"按钮；另单独展示"管理链接/口令"并提示保存。
4. 复制公开链接。
   - 检查：复制的链接指向公开团购页（不含 manageToken）。

## 流程 B · 参与者加入（对应 F2/F3）
1. 打开公开团购链接。
   - 检查：状态=OPEN 时显示报名表单；已成团/关团时显示对应状态且表单禁用。
2. 填昵称 + 份数(≥1) + 可选规格/备注 → 提交。
   - 检查：份数非正/空被拒；提交成功后本条出现在名单，进度按口径增加。
3. 页面进度更新。
   - 检查：累计与"还差 X"正确；数秒内（轮询）反映新加入。
4. 同浏览器再次打开 → 可见"我的报名"，可修改/撤销。
   - 检查：修改后合计随之变化；撤销后本条消失且进度回退；他人无法改我的（token 不匹配被拒）。

## 流程 C · 成团/关团（对应 F4）
1. 累计达到目标的那次提交完成。
   - 检查：status 变 FULFILLED，记录 fulfilledAt；页面显示"已成团 🎉"。
2. 成团后再有人尝试提交。
   - 检查：按 default 规则被拒并提示（超额是否允许=待确认）。
3. 到截止时间。
   - 检查：status 变 CLOSED；新报名被拒；汇总仍可查看。

## 流程 D · 发起人汇总（对应 F5）
1. 用管理链接/口令进入管理页。
   - 检查：显示全部报名明细 + 合计（总人数/总份数/总金额）。
2. 点"复制汇总"或"导出 CSV"。
   - 检查：内容与列表逐条一致；空报名导出仅表头不报错。
3. 无 manageToken 访问管理页。
   - 检查：被拒绝，仅公开进度可见。

## 并发验收（对应 F2/F4 边界）
- 模拟两人几乎同时提交使累计刚好达标。
  - 检查：合计不重复不丢失；status 只翻转一次；不产生超额于规则允许之外的报名。
