## F1 · 发起团购（M0）
**用户故事**：作为发起人，我想填写商品和成团条件并生成一个可分享的链接，以便把这次团购发给大家。

**验收标准（Given/When/Then）**
- Given 我在发起页，When 我填了商品名、单价、成团目标（人数或份数/金额之一）并提交，Then 系统创建团购并跳转到团购页，URL 含唯一 ID。
- Given 创建成功，When 页面加载，Then 显示商品信息、成团目标、当前进度=0、以及"复制链接"按钮。
- Given 我是发起人，When 创建成功，Then 我另外获得一个"管理链接/口令"用于后续管理（与公开链接区分）。

**边界/异常**
- 必填项为空 / 单价非正数 / 目标≤0 → 阻止提交并提示。
- 截止时间早于当前时间 → 阻止提交。

## F2 · 参与报名（M0）
**用户故事**：作为参与者，我打开团购链接，填昵称和要几份就能加入，不用注册。

**验收标准**
- Given 我打开有效团购链接且未关团，When 我填昵称 + 份数(≥1) + 可选规格/备注并提交，Then 我的报名出现在名单中，进度按份数/金额/人数增加。
- Given 我已报名，When 我在同一浏览器再次打开，Then 能看到并修改/撤销我自己的那条报名（凭本地标识，见 TECH_SPEC）。
- Given 团购已成团或已关团，When 我尝试提交，Then 被拒绝并提示当前状态。

**边界/异常**
- 份数为 0/负/非数字 → 阻止提交。
- 昵称重复 → 允许（不同人可同名），各为独立报名条目。
- 并发多人同时提交 → 合计与进度不丢单、不重复计（服务端为准）。

## F3 · 实时进度与名单（M0）
**用户故事**：作为任何访问者，我想看到当前进度和已参与名单，判断要不要加入 / 是否快成团。

**验收标准**
- Given 团购页，Then 显示：目标、当前累计（人数/份数/金额）、"还差 X"、参与名单（昵称+份数+备注）。
- Given 有人新加入，When 我停留在页面或刷新，Then 进度在数秒内反映最新值。【实时刷新方式见 TECH_SPEC，可轮询】

**边界/异常**
- 无人参与 → 显示空名单与"快来当第一个"。
- 目标达成 → 进度条满格并显示"已成团"。

---

## F4 · 成团判定与关团（M1）
**用户故事**：作为发起人，我希望达到目标或到截止时间时系统自动判定成团/关团，不用我手动盯。

**验收标准**
- Given 累计达到成团目标，When 该条报名提交完成，Then 团购状态置为"已成团"，并记录成团时间。
- Given 设置了截止时间且已到，When 有人访问或系统检查，Then 状态置为"已关团"，不再接受新报名。
- Given 发起人，When 在管理页点"手动结束/重开报名"，Then 状态相应变更（M1 可选）。

**边界/异常**
- 已成团后是否继续接受报名（超额）？→【假设：默认停止；待确认，见 GAPS】。
- 未成团即到截止时间 → 标记"未成团/已关团"，汇总仍可查看。

## F5 · 发起人汇总与导出（M1）
**用户故事**：作为发起人，我想一键拿到全部报名明细和合计，用于统一下单/收款。

**验收标准**
- Given 我持管理链接/口令进入管理页，Then 看到全部报名（昵称、份数、规格、备注、单条金额）与合计（总份数、总金额、总人数）。
- Given 管理页，When 我点"复制汇总"/"导出 CSV"，Then 得到与列表一致的文本/文件。
- Given 非管理者，When 访问管理页，Then 被拒绝（仅公开进度可见，明细/导出受限）。【假设，待确认隐私粒度】

**边界/异常**
- 空报名导出 → 生成仅表头文件，不报错。

---

## F6 · 收款信息展示（M2，非在线支付）
**用户故事**：作为发起人，我想放一个收款方式（如收款码图片或备注），让成团后大家线下付我。

**验收标准**
- Given 我在发起/管理页上传收款码图片或填收款说明，When 团购页展示，Then 成团后参与者可见收款方式。
- 明确不做在线扣款、对账、退款。
