# Feature 细节

> 客户：CMU Summer Mini Hackathon ｜ 项目：帮小区做一个团购小工具 ｜ 交付物：app（web）
> 用户故事 + 验收标准 + 边界/异常 + 优先级
> 本文件由 FDE Copilot 随每轮对话自动维护。

优先级：P0=demo 必须 / P1=增强 / P2=可选。增量切分：M0 = F1–F4，M1 = F5–F7。

---

## F1 · 团长发起团购活动 【P0 / M0】
**用户故事**：作为团长，我想快速创建一个团购活动并填入商品与规则，以便发起收单。

**验收标准**
- Given 团长在创建页，When 填写「团购标题、至少 1 个商品（名称+单价）、截止时间」并提交，Then 系统创建活动并生成唯一分享链接。
- Given 未填必填项（标题/商品/截止时间），When 提交，Then 阻止提交并高亮缺失项。
- 商品支持添加多项（名称、单价、可选单位如"箱/份"）。
- 可选字段：起送/成团份数、自提点、自提时间、团长备注。

**边界/异常**
- 单价为负或非数字 → 校验拒绝。
- 截止时间早于当前时间 → 校验拒绝。

---

## F2 · 生成并分享团购链接 【P0 / M0】
**用户故事**：作为团长，我想拿到一个可分享的链接，发到小区群让大家下单。

**验收标准**
- Given 活动创建成功，Then 展示可复制的公开链接（形如 `/g/{activityId}`）。
- 点击「复制链接」按钮，链接进入剪贴板并提示已复制。
- 任何人凭链接可访问团购详情页，无需登录。

**边界/异常**
- 访问不存在/已删除的活动 ID → 展示友好的「团购不存在或已结束」页。

---

## F3 · 居民浏览团购并下单 【P0 / M0】
**用户故事**：作为居民，我想打开链接看到团购内容，选好商品份数并留下联系方式提交订单。

**验收标准**
- Given 居民打开团购详情页，Then 看到标题、商品列表与单价、截止时间、当前进度（已订份数/参与人数）。
- Given 团购未截止，When 居民选择商品份数并填写「姓名、手机号、楼栋门牌」后提交，Then 生成一条订单，页面提示下单成功并显示订单摘要（含总金额）。
- 总金额 = Σ(商品单价 × 份数)，前端展示与后端计算一致。
- Given 团购已截止或已关闭，Then 隐藏/禁用下单入口，提示「已截单」。

**边界/异常**
- 未选任何商品（总份数=0）→ 禁止提交。
- 手机号格式非法 → 校验提示（校验规则见 GAPS Q3，默认宽松：非空即可）。
- 重复提交（同一人再次下单）→ 默认允许追加为新订单（是否合并/去重见 GAPS Q4）。

## F4 · 团长查看订单汇总与明细 【P0 / M0】
**用户故事**：作为团长，我想在后台看到所有订单的汇总和逐条明细，以便对账和备货。

**验收标准**
- Given 团长打开该活动的管理页（凭管理链接/口令，见 GAPS Q5），Then 看到：总订单数、总份数（按商品分列）、总金额。
- Then 看到逐条订单明细：下单人姓名、联系方式、门牌、所选商品与份数、单条金额、下单时间。
- 汇总数字 = 明细求和（可测：份数/金额校验一致）。

**边界/异常**
- 无订单时展示空状态。
- 未授权访问管理页 → 拒绝并提示。

---

## F5 · 截单 / 关闭团购 【P1 / M1】
**用户故事**：作为团长，我想手动截单或到期自动截单，之后不再接受新订单。

**验收标准**
- Given 团长点击「截单」，Then 活动状态变为已关闭，居民端下单入口禁用。
- Given 当前时间 ≥ 截止时间，Then 活动自动视为已截单，居民端禁止下单。

---

## F6 · 订单汇总导出（CSV）【P1 / M1】
**用户故事**：作为团长，我想把订单明细导出成 CSV，交给供货方或存档。

**验收标准**
- Given 团长在管理页点击「导出 CSV」，Then 下载包含全部订单明细列的 CSV 文件，编码兼容 Excel（UTF-8 BOM）。

---

## F7 · 自提信息与通知文案 【P1 / M1】
**用户故事**：作为团长，我想生成一段自提通知文案，方便复制到小区群。

**验收标准**
- Given 活动含自提点/自提时间，When 团长点「生成通知」，Then 输出可复制的文案（含标题、自提点、时间、总单数）。
