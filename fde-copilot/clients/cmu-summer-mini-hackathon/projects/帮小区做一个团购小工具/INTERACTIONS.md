# 交互流程与验收

> 客户：CMU Summer Mini Hackathon ｜ 项目：帮小区做一个团购小工具 ｜ 交付物：app（web）
> 逐步交互 + 每步检查/验收标准
> 本文件由 FDE Copilot 随每轮对话自动维护。

> 每步的「检查」可被自动化测试消费（Given/When/Then）。

## 流程 A · 团长发起团购（对应 F1/F2）
1. 团长进入 `/create`。
   - 检查：页面展示标题、商品（可多行增删）、截止时间等表单字段。
2. 团长填写标题、≥1 个商品（名称+单价）、截止时间，可选起送份数/自提点/时间/备注。
   - 检查：缺任一必填项，提交按钮禁用或提交被拦并高亮。
   - 检查：单价非正数或截止时间早于当前 → 报错，不提交。
3. 团长点「创建团购」。
   - 检查：返回活动详情，页面显示可复制的分享链接 `/g/{id}` 与管理链接 `/g/{id}/manage?token=...`。
4. 团长点「复制链接」。
   - 检查：剪贴板含分享链接，UI 提示「已复制」。

## 流程 B · 居民下单（对应 F3）
1. 居民打开 `/g/{id}`。
   - 检查：展示标题、商品+单价、截止时间、当前进度（已订份数 / 参与人数）。
   - 检查：活动已截单/过期 → 下单区禁用并提示「已截单」。
2. 居民为商品选择份数（可多商品）。
   - 检查：实时显示合计金额 = Σ(单价×份数)。
   - 检查：总份数为 0 时提交禁用。
3. 居民填写姓名、手机号、楼栋门牌。
   - 检查：姓名/门牌非空；手机号按规则校验（默认非空即可，规则见 GAPS Q3）。
4. 居民点「提交订单」。
   - 检查：服务端重算金额、校验活动 OPEN 且未过期，成功后返回订单摘要（商品、份数、总额）。
   - 检查：页面进度数字随之更新（已订份数 +本单份数）。

## 流程 C · 团长汇总与截单（对应 F4/F5/F6/F7）
1. 团长打开 `/g/{id}/manage?token=...`。
   - 检查：token 有效才展示，否则拒绝。
   - 检查：展示汇总（总单数、按商品分列的总份数、总金额）+ 逐条订单明细。
   - 检查：汇总数字 == 明细求和（份数、金额）。
2. 团长点「截单」。
   - 检查：活动 status→CLOSED；居民端下单入口禁用。
3. 团长点「导出 CSV」。
   - 检查：下载 CSV，含全部明细列，UTF-8 BOM，可被 Excel 正确打开。
4. （可选）团长点「生成自提通知」。
   - 检查：输出含自提点/时间/总单数的可复制文案。

## 端到端 Demo 验收（现场跑通、不改代码）
- 建团 → 复制链接 → 用另一浏览器/手机作为居民下 2 单 → 团长管理页看到 2 单且汇总正确 → 截单后居民端无法再下单。
