## 1. 问题定义
小区里的团购目前多靠微信群 + 群接龙 + 手工 Excel 汇总，存在痛点：
- 团长手工统计谁买了什么、买了几份，易漏易错；
- 居民看不到实时进度（还差几份成团、截单没）；
- 汇总、通知自提全靠人肉，效率低。

需要一个轻量 web 小工具，把「发起团购 → 居民下单 → 截单汇总 → 通知自提」这条链路数字化。

## 2. 目标（本次交付）
- **G1**：团长能 3 分钟内发起一个团购活动（商品、单价、起送/截止条件）。
- **G2**：居民无需复杂注册即可下单，实时看到团购进度。
- **G3**：团长一键查看订单汇总（谁、买了啥、几份、总金额），并能截单/导出。

## 3. 范围（M0 → M1，黑客松可 demo）
**范围内（M0 核心闭环）**
- 团长创建团购活动（含商品项、价格、截止时间/成团份数）
- 生成可分享链接
- 居民通过链接下单（选商品、填份数、留姓名+联系方式+楼栋门牌）
- 团购详情页展示实时进度（已订份数 / 参与人数）
- 团长后台查看订单汇总与明细

**范围内（M1 增强，若时间允许）**
- 截单 / 关闭团购；到达成团条件提示
- 订单汇总导出（CSV）
- 简单的自提通知（展示自提点/时间，或复制通知文案）

**范围外（本次不做，非目标）**
- 在线支付与资金结算（默认线下收款）— 除非客户明确要求（见 GAPS Q2）
- 多小区 / 多团长平台化运营、佣金分账
- 商品库存对接、供应链、物流配送
- App 原生端（仅 web）
- 复杂账号体系、实名认证

## 4. 成功指标
- **Demo 成功标准**：现场能完整跑通「团长建团 → 分享链接 → 居民下单 → 团长看到汇总」全流程，无需改代码。
- 团长发起一次团购 ≤ 3 分钟；居民下单 ≤ 3 步。
- 订单汇总数字与逐条下单记录一致（可测：金额/份数求和校验）。

## 5. 关键假设（详见 GAPS，均待客户确认）
- 采用「团长制」单团长模型，一个团购活动一个团长。
- 免支付，仅登记订单，线下收款/自提。
- 居民免注册，凭姓名+手机号+门牌下单（轻身份）。
- 部署为一个可公开访问的 web 链接（黑客松 demo）。
