## 主流程 A：诊断 → 补齐 → 投递 → 演练（单用户·格格）

### 流程 1 · 建档与诊断（对应 F1）
1. **Given** 首次进入，**When** 格格填写档案（学历/在读、研究方向、成果、语言、党员、年龄、目标地域），**Then** 档案保存成功且必填项校验通过。
2. **When** 点击"生成差距报告"，**Then** 返回含【满足项 / 缺口项 / 补齐动作+优先级 / 泰博风险应对 / 就绪度分】的报告；缺口项每条可点开看"怎么补"。
   - **检查**：报告至少覆盖「学历认证、双师型、教案/作品、简历、试讲」五类；每条缺口都有 next action。

### 流程 2 · 岗位情报（对应 F2）
3. **Given** 目标地域=东北（辽/吉/黑，城市未定则全域），**When** 刷新岗位库，**Then** 展示岗位列表，每条含学历/经验要求、是否接受海外博士（未知则标"待确认"）、编制/合同类型、截止日、来源链接、匹配标签。
   - **检查**：列表可按"匹配/存疑/不符"过滤；"匹配"岗均满足博士学历门槛；底库累计 ≥30 条；编制与合同岗均纳入（不做身份过滤）。

### 流程 3 · 造弹药（对应 F3/F4）
4. **When** 选一个目标岗点击"生成简历"，**Then** 产出定制简历初稿 + AI 评审分与改进建议；**When** 迭代，**Then** 分数可提升并留版本。
   - **检查**：简历不含任何未提供的企业/职业经历（真实性校验）。
5. **When** 指定课题（默认数字经济/电商×双创模板）生成教案/课程设计，**Then** 产出结构完整、可导出可编辑文档，并标注"AI 工具融入点"。
   - **检查**：教案含教学目标/重难点/过程/考核四要素齐全；简历 AI 评审目标分 ≥80；简历正文含"自研 agent"亮点条目；≥1 份教案以自研 agent 为案例。
5b. **When** 选一份教案点击"生成试讲脚本"，**Then** 产出说课/试讲视频脚本+分镜+讲解要点+时长配速（对齐 10–20 分钟），可导出可编辑、可迭代（对应 F8a）。
   - **检查**：脚本时长配速在 10–20 分钟区间；含分镜与逐段讲解要点；≥1 段设置"自研 agent 展示点"。
5c. **When** 点击"生成展示短视频"（默认主题=自研 FDE Copilot+Loop Engineer 系统），**Then** copilot 插件在**本地渲染出真实 MP4 成片**（LLM 脚本→程序化 motion 排版→本地 TTS→ffmpeg 合成），可换风格/改文案/调时长并版本化（对应 F8b）。
   - **检查**：成片 15–90 秒、≥1080p、含品牌 motion+逐句字幕、含 ≥1 段"自研系统"演示镜头（取自 `fde-copilot/`+`loop-engineer/` 两 submodule 真实运行/流程/代码，路径已钉死 R8；亦可取本 spec 库 `fde-copilot/clients/格格/` 的规格演进）；本地默认栈无需生成式视频大模型即可出片；任一环失败优雅降级为 5b 脚本产物；如需 AI 实拍级镜头则走外部视频模型 API adapter（Q17，客户提供 key）。
   - **检查（R7·出片位置）**：成片必落在 `<repo_root>/output/videos/<时间戳-slug>/final.mp4` 且同步到 `output/videos/latest/final.mp4`；生成结束后 UI/CLI 明确打印 `final.mp4` 的**绝对路径**，格格可直接 `open` 查看（构建前该路径无文件，属正常）。

### 流程 4 · 投递管理（对应 F5）
6. **When** 把岗位加入投递看板并更新状态，**Then** 状态机流转正确，临近截止有提醒。
   - **检查**：状态只能按 待投→已投→初筛→试讲→面试→offer/拒 合法流转。

### 流程 5 · 试讲/面试演练（对应 F6）
7. **When** 选教案进入模拟试讲，**Then** AI 评委提问+分项打分+改进建议；**When** 练高频问答（泰博/无经历/为何高职），**Then** 返回话术优化。
   - **检查**：每次演练输出可量化评分与至少 3 条可执行改进点。

### 流程 6 · 材料清单包导出（对应 F7·双交付）
8. **When** 点击"导出材料包"，**Then** 生成结构化文件夹（定制简历 docx / 教案作品集 docx-pdf / 东北岗位清单 xlsx / 试讲稿 / 求职进度 checklist / 学历认证·双师型待办清单），命名规范、可离线交付。
   - **检查**：导出内容与软件内数据一致；重复导出版本可区分、不静默覆盖旧包。
