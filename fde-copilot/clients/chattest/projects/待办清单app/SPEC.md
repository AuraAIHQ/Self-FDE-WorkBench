# 需求规格 · Spec

> 客户：chattest ｜ 项目：待办清单app ｜ 交付物：待办清单（other）
> 问题定义 / 目标 / 范围 / 成功指标 / 非目标

## 1. 问题定义
用户需要一个轻量的网页工具，随手记录待办事项，并在完成时打勾、不需要时删除。目标是「快、简单、无学习成本」。

## 2. 目标
- 提供一个可在浏览器打开即用的待办清单页面。
- 支持三个核心动作：新增任务、标记完成/取消完成、删除任务。
- 「简单」是第一原则：无注册、无引导、打开即用。

## 3. 范围（In Scope）
- 单页 Web 应用（桌面 + 移动浏览器自适应）。
- 任务列表的增 / 改（完成状态）/ 删。
- 【调研假设·待确认】数据持久化到浏览器本地存储（localStorage），刷新/关闭后不丢失，单机单人使用。

## 4. 非目标（Out of Scope，本期不做）
- 用户账号、登录、多用户协作。
- 跨设备同步、云端后端存储。
- 任务的截止日期、提醒、优先级、标签、分类、子任务、拖拽排序。
- 搜索、筛选、批量操作。
> 以上非目标为当前假设的边界，若客户确认需要可提级为 M1+。

## 5. 成功指标
- 用户可在 3 秒内完成「新增一条任务」的操作，无需任何说明。
- 三个核心动作（增/完成/删）功能均通过验收标准（见 FEATURES.md / INTERACTIONS.md）。
- 刷新页面后任务列表与完成状态保持不变（若持久化假设成立）。

## 6. 待确认的范围问题
见 GAPS.md：数据是否只需本地保存、是否需要任务的额外属性、完成任务的展示方式。
