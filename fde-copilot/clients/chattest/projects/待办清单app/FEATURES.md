# Feature 细节

> 客户：chattest ｜ 项目：待办清单app ｜ 交付物：待办清单（other）
> 用户故事 + 验收标准 + 边界/异常 + 优先级

## 优先级与增量划分
- **M0（本期核心，必须交付）**：F1 新增任务、F2 标记完成/取消、F3 删除任务、F4 本地持久化。
- **M1（待客户确认后再排）**：编辑任务文本、清除所有已完成、任务额外属性（截止日期/优先级）。

---

## F1 · 新增任务 —— 【M0】
**用户故事**：作为用户，我想在输入框输入内容并提交，让它作为一条待办出现在列表里，这样我能记录要做的事。

**验收标准（Given/When/Then）**
- Given 页面已打开且输入框为空，When 用户输入「买牛奶」并按回车（或点「添加」按钮），Then 列表中新增一条文本为「买牛奶」、状态为未完成的任务，且输入框被清空。
- Given 输入框内容为空或仅含空白字符，When 用户提交，Then 不新增任务（忽略提交），输入框保持聚焦。
- 【调研假设·待确认】新任务插入在列表**顶部**（最新在上）。
- Given 输入超长文本（如 >200 字），When 提交，Then 任务正常创建并在列表中换行完整显示（不截断、不溢出破坏布局）。

**边界/异常**
- 前后空白自动去除（trim）后再判断是否为空。
- 允许重复文本的任务（不做去重）。
- 文本按纯文本渲染，禁止 HTML 注入（防 XSS）。

---

## F2 · 标记完成 / 取消完成 —— 【M0】
**用户故事**：作为用户，我想点击任务的勾选框来标记它已完成或恢复未完成，这样我能追踪进度。

**验收标准（Given/When/Then）**
- Given 一条未完成任务，When 用户点击其勾选框，Then 该任务变为已完成状态：勾选框呈选中态，任务文本显示删除线（划线）。
- Given 一条已完成任务，When 用户再次点击勾选框，Then 恢复为未完成状态：去除选中态与删除线。
- 状态变更立即持久化，刷新后保持。

**边界/异常**
- 【调研假设·待确认】已完成任务仍留在原列表中（划线保留），不隐藏、不移到单独区域。—— 展示方式待客户确认。

---

## F3 · 删除任务 —— 【M0】
**用户故事**：作为用户，我想删除某条任务，这样列表只保留我关心的事。

**验收标准（Given/When/Then）**
- Given 列表中有若干任务，When 用户点击某条任务的删除按钮，Then 该任务从列表中移除，且不影响其他任务的顺序与状态。
- 删除立即持久化，刷新后该任务不再出现。
- Given 删除最后一条任务后列表为空，Then 显示空状态提示（如「还没有任务，添加一条吧」）。

**边界/异常**
- 【调研假设·待确认】删除**无二次确认**（简单优先，删错就重新添加）。若客户希望防误删可加确认或撤销，进 M1。

---

## F4 · 本地持久化 —— 【M0，假设待确认】
**用户故事**：作为用户，我关掉网页再打开时，希望之前的任务和完成状态还在。

**验收标准（Given/When/Then）**
- Given 用户已添加若干任务并标记了部分完成，When 刷新页面或重新打开该网址，Then 列表内容、顺序、完成状态与关闭前完全一致。
- Given 首次访问（无历史数据），Then 显示空状态，不报错。

**边界/异常**
- 存储损坏/无法解析时，安全回退为空列表，不白屏报错。
- 【调研假设·待确认】使用浏览器 localStorage，即数据仅存于当前浏览器、不跨设备。是否需要跨设备同步 → 见 GAPS。
