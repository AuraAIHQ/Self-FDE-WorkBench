# 交互流程与验收

> 客户：chattest ｜ 项目：待办清单app ｜ 交付物：待办清单（other）
> 逐步交互 + 每步检查/验收标准（可被自动化测试消费）

## 流程 A · 新增任务
1. 打开页面 → **检查**：显示标题、空输入框、空状态提示、输入框自动聚焦。
2. 在输入框输入「买牛奶」→ **检查**：输入框显示「买牛奶」。
3. 按回车（或点「添加」）→ **检查**：列表顶部出现一行「买牛奶」，勾选框未选中，输入框清空并保持聚焦，空状态提示消失。
4. 不输入任何内容直接按回车 → **检查**：列表数量不变，无新增空任务。
5. 输入「  （仅空格）」提交 → **检查**：不新增任务。

## 流程 B · 标记完成 / 取消
1. 已有一条未完成任务「买牛奶」。
2. 点击其勾选框 → **检查**：勾选框变选中态，「买牛奶」文本出现删除线。
3. 刷新页面 → **检查**：该任务仍为已完成（划线）状态。
4. 再次点击勾选框 → **检查**：去除选中态与删除线，恢复未完成。

## 流程 C · 删除任务
1. 列表中有「买牛奶」「交房租」两条任务。
2. 点击「买牛奶」的删除按钮 → **检查**：「买牛奶」消失，「交房租」仍在且状态不变。
3. 再删「交房租」→ **检查**：列表为空，显示空状态提示。
4. 刷新页面 → **检查**：列表仍为空（删除已持久化）。

## 流程 D · 持久化回访
1. 添加「A」「B」「C」三条，标记「B」完成。
2. 关闭标签页并重新打开同一网址 → **检查**：列表按原顺序显示 A、B、C，其中 B 为已完成状态。

## 流程 E · 异常/边界
1. 手动损坏 localStorage 数据后打开页面 → **检查**：页面正常渲染为空列表，不白屏、不报错。
2. 输入 `<script>alert(1)</script>` 作为任务并提交 → **检查**：作为纯文本显示，不执行脚本。
