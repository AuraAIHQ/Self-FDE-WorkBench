# 需求规格 · Spec

> 客户：chattest ｜ 项目：dir1 ｜ 交付物：web（待办清单）

---

## 一句话定位
一个极简的待办清单网页，用来快速记录、标记完成和清理任务。

## 目标用户
需要快速记录碎片任务的个人用户。【假设·待确认】

## 核心功能

| 功能 | 描述 | 验收标准 |
|------|------|----------|
| 添加任务 | 输入文字后回车或点击按钮添加一条任务 | 输入非空文字能出现在列表中 |
| 标记完成 | 点击任务前的复选框标记完成状态 | 任务文字出现删除线/变灰 |
| 删除任务 | 点击删除按钮移除任务 | 任务从列表中消失 |

## 范围

**范围内**
- 单页面纯前端交互
- 添加/完成/删除三个操作

**范围外**
- 任务编辑/重命名
- 多人协作
- 账号体系
- 数据持久化（【待确认·q1】）

## 技术方向

- **前端**：纯 HTML + CSS + JavaScript，单文件实现【假设·待确认】
- **数据存储**：
  - 若无需持久化 → 内存变量
  - 若需持久化 → localStorage【假设·待确认】
- **UI 框架**：无，纯原生实现（保持轻量）【假设·待确认】
