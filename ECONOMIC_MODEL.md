# 可持续经济模型 · Self-FDE-WorkBench

> 活文档。**「构想」= 团队定位;「建议」= 待一起厘清的取舍。**
> 关联：[MISSION](../Brood/protocol/MISSION.md)（数字公共物品 / 数字主权）· PGL（AgentStore 分账）· [DEPLOY.md](./DEPLOY.md)

## 1. 定位（不是工具，是「造工具的工作台」）

WorkBench 是一个 **AI-native 的工作坊**：普通人只要**开口说痛点**，`fde-copilot` 持续追问 + 替他调研分析，把真实场景提炼成技术规格；`loop-engineer` 据此把**专属工具**造出来、交付；`capability-packs` 提供配置一次即可复用的**原子能力**。

- 服务对象：**个体 / 家庭 / 社区 / 企业**——不管从哪个视角,只要会说话就行。
- 交付物是**工具**（在线工具 / 可安装应用 / 自动化 / agent），一次性产物（PPT/视频/报告）只是最浅的子集。
- 价值内核：**个性化**——因为是他自己的场景，他最懂,工具也最贴他的日常。

## 2. 部署形态与数据主权

**个体/家庭：一台 Mac Mini 为一个家庭服务**（核心按 2 人算，可含孩子/父母）。

- **所有数据/信息存在这台 Mac Mini 上**，加密 + 安全措施（FileVault / 凭证 chmod 600 / Cloudflare Access，见 DEPLOY.md）。
- 定制的工具**也跑在这台 Mac Mini 上**。所谓「在线」不是网页 SaaS，而是 **网络随时可触达你自己的 Mac Mini**（Cloudflare Tunnel）——可 24/7 跑任务、自动响应。
- **造**的过程一定能在 Mac Mini 上完成；**跑**的形态可变。

**企业：** 造的过程仍在 Mac Mini 上完成，但**部署形态通常不在 Mac Mini**（上企业自己的云/机房）。

**这条与使命一致：** 数据不出本地 = 数字主权;科技红利归普通人。

## 3. 开源核心 + 三条收入线（Linux/Red Hat 式）

**核心全开源**——你可以自己搞一台 Mac Mini、跑起来、完全自持。我们的利益是**让普通人得到数字生活的福祉**，不是靠锁定赚钱。在此之上提供**可选**的付费服务：

| # | 收入线 | 卖什么 | 收费口径 | 特征 |
|---|---|---|---|---|
| A | **Workbench 租用** | 借我们的 AI-native 工作坊「造」工具 | 折旧摊销 + 可持续成本 → 折算为 **token + 计算单元(CU)** | 个人按小时/按项目;可长期租一台专属 Mac Mini |
| B | **部署/托管服务** | 帮你把造好的工具跑起来、维护、接 token | 时间 + token 通道 | 低但**稳定**的长期收入 |
| C | **企业咨询** | 复杂商业场景的洞察 + 定制 | 项目制 | 高触达、高毛利，早期现金流 |

**自持路径永远存在**：不想付费 → 开源自部署。付费买的是**便利、可靠、专业**，不是准入。

## 4. 计价基础：两个计量表（token + 计算单元）

产品里已经在实时统计**两种单位**（见各应用用量面板）：

- **Token**：模型 API 消耗（输入/输出/缓存）——对应「造」和「跑」里调模型的部分。
- **计算单元 CU（墙钟计算秒）**：机器/电力/时间占用——对应 Mac Mini 折旧 + 运行时间。

**建议的收费结构（见 §6）**：token 走**透传或小加价**，CU 走**按时/按机**定价。两个成本驱动分开计,避免把固定的硬件折旧硬摊到浮动的 token 上。

## 5. 与 Mycelium / PGL 的对接（构想）

- `capability-packs` 的原子能力 = **AgentStore / PGL** 的分发对象;造出的专属工具可上架。
- 收益按 PGL 三角色分账：**Supplier 50–90% / Wrapper 0–40% / Seller 固定 10%**，商业收入 **40% 回流社区国库**。
- 这样「造工具的工作台」不是独立商业，而是**数字公共物品**生态的一环。

## 6. 待一起厘清的问题 / 建议（中立客观）

> 愿景站得住、且与使命一致。以下不是反对，是把「更清晰更可靠」落到取舍上。

1. **「专属数据在你的 Mac Mini」与「按小时便宜租用」有张力。**
   - 便宜按小时 → 暗示**共享多租户** Mac Mini（数据隔离/安全更难，主权故事变弱）。
   - 你的数据在你自己的机器 → 暗示**专属** Mac Mini（主权强，但贵、难按小时）。
   - **建议**：**「造」在共享/我们的 Mac Mini 上（临时、用后擦除）;「跑 + 数据」落到你专属的 Mac Mini。** 主权活在「部署」环节,「造」环节可共享摊薄成本。

2. **按小时租用可能低估了「追问」的时间维度。**
   - fde-copilot 的价值是**持续追问**，复杂场景的提炼是多轮、跨天的关系,不是一次性。
   - **建议**：个人侧更接近**按项目（按结果）**或**低月费保留工作坊**，而非纯按小时;小时费适合轻量一次性活。

3. **两个成本驱动，两个表分开计（别混）。**
   - Token（模型，浮动）和硬件折旧/电（固定/按时）是不同性质。全摊进「token」会在高/低负载下错价。
   - **建议**：token 透传（或小加价）+ CU 按时定价。产品里已有这两个表,天生支持。

4. **维护是「沉默成本」，托管收入必须覆盖它。**
   - 「只为你部署的工具」会因模型/依赖/平台 API 变化（如小红书 Cookie 过期）需要**持续人工维护**，不只是服务器时间。
   - **建议**：把托管定价里显式含**维护人力**,否则规模化后是亏损项。

5. **全开源下，护城河要说清（否则像把家底送人）。**
   - 代码开源后,壁垒不在代码,在于：① `capability-packs` 库的**广度 + 策展**;② 累积的场景/eval 让 fde-copilot **洞察越来越准**;③ 托管的**可靠性**。
   - **建议**：文档明确「我们的护城河是能力库 + 上下文积累 + 托管可靠性」,让开源成为增长引擎而非放弃收入。

6. **咨询（C）和产品（A/B）是两种生意,别混算账。**
   - 咨询是人头绑定、高触达;适合做**楔子和早期现金流**,但毛利模型和产品不同。
   - **建议**：对齐 Mycelium 的「开源组织 + 商业实体（HyperCapital）」双生结构,咨询/托管走商业实体、40% 回流,产品核心留在公共物品层。

---

## 一句话总结

**开源的、数据主权的、为普通人造专属 AI 工具的工作台;靠「造(租用)+ 跑(托管)+ 企业咨询」三条线可持续,以 token + 计算单元透明计价,并入 Mycelium/PGL 的公共物品与分账体系。** 愿景可靠;成败在于把「造→部署→维护」做到足够便宜且足够窄,以及厘清共享/专属、按项目/按时、两表计价这几处取舍。
